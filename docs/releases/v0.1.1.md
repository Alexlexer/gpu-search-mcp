# gpu-search-mcp v0.1.1

> **Status:** Unreleased — release preparation in progress.

## Summary

Maintenance release adding the repository signal scan surface and Apple Silicon MPS device support. No breaking changes to existing API or MCP behavior. All v0.1.0 integrations continue to work without modification.

---

## New features

### Repository signal scan (`POST /scan/signals`)

A new endpoint and MCP tool that scans an indexed repository for audit signals — patterns that commonly indicate legacy patterns, risky constructs, or areas requiring attention in a codebase.

**22 built-in signals across 7 categories:**

| Category | Signals |
|---|---|
| `legacy-dotnet` | `web.config`, `packages.config`, `app.config`, `global.asax`, `asmx`, `aspx` |
| `config` | `appsettings.json`, `connectionStrings`, `hardcoded-ip`, `hardcoded-port` |
| `sql` | `raw-sql`, `stored-proc-exec`, `dynamic-sql`, `datatable` |
| `async-risk` | `task-result`, `task-wait`, `getawaiter-getresult`, `async-void` |
| `exception-risk` | `catch-all`, `empty-catch`, `swallowed-exception` |
| `di` | `service-locator`, `static-instance` |
| `tests` | `no-test-files` |

**HTTP endpoint:**

```bash
POST /scan/signals
Content-Type: application/json

{"root": "/path/to/repo"}   # optional — uses indexed root if omitted
```

**MCP tool:**

```
scan_repository_signals(root="/path/to/repo")
```

See `docs/signal-scan.md` for the full signal catalog and LegacyLens integration guide.

### Apple Silicon MPS device support

Centralized torch device resolution selects the best available backend automatically:
**CUDA > MPS > CPU**

**CLI flag:**

```bash
gpu-search-mcp --device auto       # default: cuda > mps > cpu
gpu-search-mcp --device mps        # force Apple Silicon GPU
gpu-search-mcp --device cuda       # force NVIDIA CUDA
gpu-search-mcp --device cpu        # force CPU
```

**Environment variable (overrides flag):**

```bash
GPU_SEARCH_DEVICE=mps gpu-search-mcp --directory /path/to/repo --http
```

**Device metadata in `/health` and `/stats`:**

```json
{
  "device": {
    "backend": "cuda",
    "torchDevice": "cuda",
    "reason": "CUDA available (auto)",
    "warnings": []
  }
}
```

If MPS is requested but the sentence-transformers model fails to load on MPS (common on some PyTorch/macOS version combinations), the server retries on CPU with a logged warning. Search behavior is identical.

## LegacyLens audit integration note

`POST /scan/signals` is designed as the primary audit feed for LegacyLens. The response is structured JSON:

```json
{
  "root": "/path/to/repo",
  "totalMatches": 14,
  "categories": {
    "legacy-dotnet": {
      "signals": [
        {
          "id": "web.config",
          "label": "web.config reference",
          "description": "...",
          "matches": [
            {
              "file": "src/App.csproj",
              "absoluteFile": "/path/to/repo/src/App.csproj",
              "lineStart": 12,
              "lineEnd": 12,
              "snippet": "<Content Include=\"web.config\" />"
            }
          ]
        }
      ]
    }
  }
}
```

Consume `categories[category].signals[].matches` for file-level signal attribution. `totalMatches` gives a quick repository-level count without iterating the full tree.

## Apple Silicon MPS note

MPS acceleration is supported for both pattern search (PyTorch tensor ops) and semantic embedding (sentence-transformers). If the embedding model fails to load on MPS — which can happen on some PyTorch/macOS version combinations — the server automatically retries on CPU and logs a warning. Pattern search is unaffected by embedding fallback.

To verify MPS resolution locally:

```bash
python -c "from gpu_service.device import resolve_torch_device; print(resolve_torch_device('auto'))"
```

## Known limitations

All v0.1.0 limitations carry forward:

- **Dependency analysis is heuristic, not compiler-accurate.** Regex-based graph; misses dynamic imports and generated code.
- **Semantic model requires download on first use.** BAAI/bge-small-en-v1.5 (~90 MB) from HuggingFace. Set `HF_HUB_OFFLINE=1` if the model is already cached.
- **Signal scan matches file content, not file names.** Signals like `web.config` fire on content that references those names, not on the config files themselves.
- **Signal scan is heuristic.** Pattern-based matching — not a static analyzer. Expect some false positives and false negatives.
- **HTTP mode has no authentication.** Designed for localhost or Tailscale-only access. Do not expose to the public internet.
- **Secret redaction is best-effort.** Catches common patterns but is not a DLP scanner.

## Upgrade notes from v0.1.0

No migration steps required. Upgrade by pulling the latest code:

```bash
git pull origin master
pip install -e ".[test,ast]"
```

Verify the upgrade:

```bash
pytest tests/ -v --timeout=60 --ignore=tests/test_thread_safety.py -k "not with_semantic"
python scripts/smoke_test.py
```

The server can be restarted in place — all caches remain valid.
