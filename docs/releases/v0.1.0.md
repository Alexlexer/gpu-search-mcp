# gpu-search-mcp v0.1.0

> **Status:** Unreleased — release preparation in progress.

## Summary

First tagged release of gpu-search-mcp. The server is a working prototype used daily for
GPU-accelerated code search over large local repositories. Core pattern search, semantic
search, dependency analysis, and the HTTP API are stable enough for daily use and local
integrations such as LegacyLens.

---

## Features

### GPU pattern search

Loads every indexed source file into VRAM as a single concatenated byte corpus and runs exact
queries as vectorized CUDA operations. No custom kernels — pure PyTorch. Falls back to CPU
automatically when no GPU is present.

- 10–15× faster than ripgrep (warm) on a 285 MB repo (RTX 4060, CUDA).
- Sub-15 ms per query after the ~3 s startup index build.
- Persistent cache under `.gpu-search-cache/` — subsequent starts skip the rebuild if nothing
  changed.

### Semantic search

Embeds source files with BAAI/bge-small-en-v1.5 (90 MB, 384 dims). Queries run as a single
GPU matmul. Cache persists across restarts (~3 s reload).

- Finds code by meaning, not just exact tokens.
- `auto` mode routes prose queries to semantic and identifier queries to pattern.
- `hybrid` mode runs both in parallel and merges results.

### Dependency analysis

Parses import statements with per-language regex heuristics and builds a sparse in-VRAM graph.
Answers "what transitively imports this file?" before you edit it.

- Supports Python, JS/TS, Go, Rust, Java, C#, Ruby.
- C# extras: `using` imports, namespace → file mapping, base/interface type → file mapping.
- **Heuristic, not compiler-accurate** — see Known Limitations.

### C# AST support

- Tree-sitter block expansion when `tree-sitter-c-sharp` is installed.
- Brace-matching fallback for common class/method/controller-action blocks when it is not.
- Skeleton mode folds unexpanded function bodies to `...  # N lines`.

### HTTP mode

Exposes all search and read tools as a local JSON API. Designed for integrations like
LegacyLens that cannot use MCP stdio transport.

```bash
gpu-search-mcp --directory /path/to/repo --http --port 8765
```

All endpoints return both a backward-compatible `result` string and stable structured JSON
fields. The structured fields are the intended integration surface for API clients.

### LegacyLens integration

The structured HTTP response fields (`results`, `file`, `absoluteFile`, `lineStart`, `lineEnd`,
`content`, `language`, `impactedFiles`) are the stable surface for LegacyLens. The `result`
string is kept for backward compatibility and will not be removed.

## Security notes

- **HTTP binds to `127.0.0.1` by default.** Pass `--host 0.0.0.0` only if you understand the
  risks and have network-level controls in place. Do not expose this API to the public internet.
- **Path validation:** all HTTP file endpoints reject paths outside the indexed roots. Path
  traversal (`../`) and absolute paths to other directories return HTTP 400.
- **`.env` exclusion:** `.env` files are not indexed by default. Pass `--allow-env-files` to
  opt in — a warning is printed when this flag is active.
- **Secret redaction:** common credential patterns (API keys, bearer tokens, passwords,
  connection strings, PEM keys, AWS access key IDs) are replaced with `[REDACTED]` in all
  search output. This is best-effort pattern matching — do not rely on it as your only secret
  protection.
- Use Tailscale or local firewall rules if you need to access the HTTP API from another machine
  on the same network.

---

## Known limitations

- **Dependency analysis is heuristic, not compiler-accurate.** The graph is built with
  regex/language-specific rules and will miss dynamic imports, conditional imports, and
  generated code. Use `dep_impact` results as a starting point, not a guarantee.
- **Semantic model may need download/cache.** The BAAI/bge-small-en-v1.5 model (~90 MB)
  is downloaded from HuggingFace on first use. This requires internet access or a pre-cached
  HuggingFace download. Set `HF_HUB_OFFLINE=1` if you have the model cached locally.
- **GPU acceleration depends on local backend availability.** CUDA is used when an NVIDIA GPU
  is present; Apple MPS is used on supported Macs; otherwise CPU is used automatically. CPU
  fallback works but is slower — pattern search is still fast; semantic embedding is noticeably
  slower for large repos.
- **Do not expose HTTP mode publicly.** The API has no authentication. It is designed for
  localhost or Tailscale-only access.
- **Secret redaction is best-effort.** It catches common patterns but is not a DLP scanner.
  Do not index repos containing production secrets.

## Install

```bash
# Clone and install with all extras
git clone https://github.com/Alexlexer/gpu-search-mcp.git
cd gpu-search-mcp
pip install -e ".[test,ast]"

# Or with uv
uv venv --python 3.12
uv sync --extra ast
```

Optional Tree-sitter grammars (Python, TypeScript, C#) improve AST block expansion and
skeleton mode:

```bash
pip install -e ".[ast]"
```

## HTTP mode quick start

```bash
# Start the server in HTTP mode (indexes current directory by default)
gpu-search-mcp --directory /path/to/repo --http --port 8765

# Health check
curl http://127.0.0.1:8765/health

# Search
curl -X POST http://127.0.0.1:8765/search/code \
  -H "Content-Type: application/json" \
  -d '{"query": "UserService", "mode": "pattern", "contextMode": "compact"}'

# Read a block
curl -X POST http://127.0.0.1:8765/read/block \
  -H "Content-Type: application/json" \
  -d '{"filepath": "/path/to/repo/src/UserService.cs", "line": 42}'

# Dependency impact
curl -X POST http://127.0.0.1:8765/dependency/impact \
  -H "Content-Type: application/json" \
  -d '{"filepath": "/path/to/repo/src/UserService.cs"}'
```

## LegacyLens integration note

Consume the structured fields, not the `result` string:

| Field | Type | Description |
|---|---|---|
| `results[].file` | string | Relative path from indexed root |
| `results[].absoluteFile` | string | Absolute path |
| `results[].lineStart` | int | Start line (1-indexed) |
| `results[].lineEnd` | int | End line (1-indexed) |
| `results[].score` | float | Relevance score |
| `results[].snippet` | string | Redacted code snippet |
| `results[].engine` | string | `"pattern"` or `"semantic"` |
| `file` (read endpoints) | string | Relative path |
| `absoluteFile` (read) | string | Absolute path |
| `content` (read) | string | Redacted file/block content |
| `language` (read) | string | e.g. `"csharp"`, `"python"` |
| `impactedFiles[].hops` | int | Import depth (1 = direct) |

The `result` string will remain in every response. It is not the stable surface.
